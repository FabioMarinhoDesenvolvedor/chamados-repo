## Segurança (revisão completa nesta sessão)
- **Corrigido — vazamento de `passwordHash`**: detalhe do chamado e resposta de
  comentário expunham o hash de `requester`/`assignee`/`author`. Agora passam por
  `toUserPublic`. (commit b5fdecb)
- **Hardening** (commit 7856b46):
  - `JWT_SECRET` obrigatório e forte (helper `getJwtSecret`, fail-fast; rejeita o
    placeholder). O segredo do `.env` de DEV foi rotacionado p/ valor forte — sessões
    antigas invalidadas (relogar). `.env` é gitignored.
  - `helmet` (nosniff, X-Frame-Options, remove X-Powered-By) + `nosniff` no endpoint
    de anexos.
  - Rate limit (`@nestjs/throttler`) em `/auth/*`: 10 req/min por IP.
- **Sem SQL injection** (tudo Prisma parametrizado) e **sem command injection** no
  backup (`spawn` com array de args, sem shell). Autorização: todo endpoint admin com
  `@Roles('ADMIN')`; IDOR barrado por `ensureCanView`.
